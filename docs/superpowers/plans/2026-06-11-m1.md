**Goal:** Working diffler M1: neogit-style TUI (status/log/diff, stage/unstage/discard/
commit/branch), comment-based review (visual mode, viewed marks, markdown export), live
watcher, embedded MCP server, fixture+PTY+MCP tests, publish workflows, README.

**Architecture:** `diffler-core` pure logic behind a `Vcs` trait (git2 impl only); binary
hosts TUI + MCP + watcher on one tokio runtime. No code above the trait imports git2.

**Gates for every task:** TDD, `just ci` green before each commit, snapshot tests for every
new screen state (insta review rule: read `.snap.new` before accepting), short imperative
commit messages, no AI-slop comments.

## Phase C â€” MCP server

### C1. Tools + integration
`src/mcp.rs`: rmcp `StreamableHttpService` nested in axum at `/mcp`, port from `--port`

directions (MCP mutations â†’ TUI redraw). Tests: spawn server on ephemeral port in a tokio
test, drive with `rmcp` client over streamable HTTP: full loop add-comment(TUI side via
state handle) â†’ get_comments â†’ reply â†’ session reflects Replied â†’ wait_for_feedback
