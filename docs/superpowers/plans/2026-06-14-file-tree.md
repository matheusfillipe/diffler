# Collapsible file tree (neo-tree style) for sidebar + status sections

Replace the flat full-path file lists with a collapsible directory tree, in both
the diff-view sidebar and the status screen sections. Basenames only, folders
toggle open/closed, per-file status glyph + `+N -M` diffstat. Like neovim
neo-tree (see the user's reference image).

## Shared tree module — `crates/diffler/src/tree.rs` (pure, unit-tested)

Build a directory trie from an ordered file list and flatten it to visible rows
honoring fold state. No rendering, no app state — just structure + navigation
math, fully unit-testable.

```rust
pub enum TreeNode {
    Dir { path: String, name: String },   // path = full dir path (fold key); name = last segment
    File { index: usize, name: String },  // index into the source file slice; name = basename
}
pub struct TreeRow { pub depth: usize, pub node: TreeNode }

/// Build the flattened, fold-respecting visible rows for `paths` (each path is
/// a file's repo-relative path, in input order). `folded` holds folded dir
/// paths. Directories are sorted before files at each level; entries keep input
/// order within a kind. Default (a dir not in `folded`) is expanded.
pub fn visible_rows(paths: &[&str], folded: &BTreeSet<String>) -> Vec<TreeRow>;
```

- Each path component below the last is a `Dir`; the last is a `File` carrying
  its index into `paths`. A `Dir` row appears once even when many files share it.
- A folded dir contributes its own row but none of its descendants.
- Collapse single-child dir chains neo-tree style: `a/b/c/d` where each level
  holds exactly one subdirectory becomes a single `a/b/c/d` row (fold key = the
  deepest dir path, display name = the joined segments). A dir holding a file or
  more than one child stops the chain. Multi-child dirs stay nested.
- Unit tests: nested paths produce the expected (depth, node) sequence; folding
  a dir hides its subtree; sibling order (dirs first, then files, stable);
  root-level files (no dir) at depth 0; a single file; identical-basename files
  in different dirs.

## Diff-view sidebar (the "viewer") — app/diff.rs + ui/diff.rs

The sidebar becomes the tree over `model.files`.

- `DiffView` gains `folded_dirs: BTreeSet<String>` (per-view, persists across
  refresh) and a `tree_cursor: usize` (index into the current visible tree
  rows). `selected: usize` (the file shown in the pane) stays, derived from the
  file under `tree_cursor` when it's a File row.
- Navigation (focus = List): `j`/`k` move `tree_cursor` over visible rows (dirs
  and files); when the cursor is on a File row, that file becomes `selected` and
  the pane updates live. On a Dir row the pane keeps the last file. `<cr>`/`za`
  on a Dir toggles its fold; `<cr>` on a File focuses the Diff pane (unchanged).
  `gg`/`G` first/last visible row. `<c-n>`/`<c-p>` jump to next/prev FILE
  (skipping dir rows), keeping focus — the quick file switch.
- `v` (mark viewed) advances to the next unviewed FILE row as today.
- Render (`draw_sidebar`): one line per visible row, indented `depth * 2`. Dir
  rows: `▸`/`▾` + dir name in dim/fg. File rows: status glyph (colored) +
  basename + ` ✓` + ` ·N` + the `+A -B` diffstat (reuse `diffstat_spans`; drop
  if the sidebar is too narrow — keep glyph+name+marks first, append stat only
  if it fits). Selected (tree_cursor) row gets the `▌` marker + cursor-line bg.
  No full paths anywhere.

## Status sections — app/status.rs + ui/status.rs

Each section (Untracked/Unstaged/Staged) lists its files as a tree under the
header; inline hunk-expand still works on a file row.

- Per-section `folded_dirs: [BTreeSet<String>; 3]` (alongside the existing
  per-section file-expand + enriched sets). Section fold (the header `▸/▾`) is
  unchanged and sits above the tree.
- The flattened `Row` model gains dir rows: extend `Row` with
  `Dir { section, path }` (or similar) between `SectionHeader` and `File`. The
  cursor navigates headers → dir rows → file rows → (inline) hunk rows. `<tab>`
  on a Dir row folds that dir; on a file row expands its inline diff (today's
  behavior); on a section header folds the section (today's behavior).
- Render file rows with basename + indent (depth within the section), status
  glyph colored, `+A -B` (already added). Dir rows: `▸/▾` + name, indented.
  Drop the full path (today's `file_spans` shows the whole path — switch to
  basename + tree indent).
- Recent commits section unchanged (not a file tree).

## Notes

- "We don't care about non-git-tracked files" — untracked files still appear as
  normal changes in their section's tree (they're real changes to review); no
  special handling, just don't crash on them. Confirm untracked files slot into
  the tree by their path like any other.
- Diffstat per file is already available via `FileDiff::diffstat()`.
- Keep `diff_render`, comments, visual-select, viewed, MCP, feedback untouched —
  this is a file-list presentation + navigation change only.
